# Исправление ошибки ERR_CERT_DATE_INVALID

## Причина
Ошибка возникает при загрузке внешних ресурсов (Google Fonts, Bootstrap CDN) из-за проблем с SSL-сертификатом.

## Решения

### 1. Проверить системное время (САМОЕ ЧАСТОЕ)
```
1. Откройте Параметры Windows → Время и язык
2. Включите "Установить время автоматически"
3. Нажмите "Синхронизировать сейчас"
4. Перезапустите браузер
```

### 2. Игнорировать (для разработки)
Ошибка не критична — ресурсы всё равно загружаются. Можно просто игнорировать в консоли.

### 3. Использовать локальные файлы

#### Скачать Bootstrap 5.3.3
```bash
# Скачайте с https://getbootstrap.com/docs/5.3/getting-started/download/
# Распакуйте в static/bootstrap/
```

#### Скачать Bootstrap Icons
```bash
# Скачайте с https://icons.getbootstrap.com/
# Распакуйте в static/bootstrap-icons/
```

#### Обновить base.html
Замените CDN-ссылки на локальные:

```html
<!-- Вместо CDN -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- Используйте локальные -->
<link href="{% static 'bootstrap/css/bootstrap.min.css' %}" rel="stylesheet">
```

### 4. Очистить кэш браузера
```
Chrome: Ctrl+Shift+Delete → Очистить кэш
Edge: Ctrl+Shift+Delete → Очистить кэш
```

### 5. Попробовать другой браузер
Если проблема только в одном браузере — используйте другой для разработки.

## Проверка
После исправления откройте консоль браузера (F12) и проверьте, что ошибок нет.
